## 关于 mybatis 使用 lombok的@Builder导致 Cannot determine value type from string 的报错排查

这是由于 DefaultResultSetHandler.createResultObject 方法，使用了不正确的结果解析处理方法导致的。 代码片段

```java
public class DefaultResultSetHandler implements ResultSetHandler {
    private Object createResultObject(ResultSetWrapper rsw, ResultMap resultMap, List<Class<?>> constructorArgTypes, List<Object> constructorArgs, String columnPrefix) {
        final Class<?> resultType = resultMap.getType();
        final MetaClass metaType = MetaClass.forClass(resultType, reflectorFactory);
        final List<ResultMapping> constructorMappings = resultMap.getConstructorResultMappings();
        if (hasTypeHandlerForResultObject(rsw, resultType)) {
            return createPrimitiveResultObject(rsw, resultMap, columnPrefix);
        } else if (!constructorMappings.isEmpty()) {
            // 没有构造方法
            return createParameterizedResultObject(rsw, resultType, constructorMappings, constructorArgTypes, constructorArgs, columnPrefix);
        } else if (resultType.isInterface() || metaType.hasDefaultConstructor()) {
            // 无参构造方法
            return objectFactory.create(resultType);
        } else if (shouldApplyAutomaticMappings(resultMap, false)) {
            return createByConstructorSignature(rsw, resultType, constructorArgTypes, constructorArgs);
        }
        throw new ExecutorException("Do not know how to create an instance of " + resultType);
    }
}

```

这个地方或根据对象的构造参数情况去进行不同的处理。

当在DO中使用 @Builder 注解时候，而提供无参构造函数，会导致该问题出现。

错误使用方法：

```java

@Getter
@Setter
@Builder
@TableName("sys_person")
@ApiModel(value = "SysPerson对象", description = "个人信息表")
public class SysPerson extends BaseEntity {

    private static final long serialVersionUID = 1L;

    @ApiModelProperty("姓名")
    private String name;

    @ApiModelProperty("出生日期")
    private LocalDateTime birth;

    @ApiModelProperty("身份证号码")
    private String idCard;


}
```

编译过后的文件

```java

@TableName("sys_person")
@ApiModel(
        value = "SysPerson对象",
        description = "个人信息表"
)
public class SysPerson extends BaseEntity {
    private static final long serialVersionUID = 1L;
    @ApiModelProperty("姓名")
    private String name;
    @ApiModelProperty("出生日期")
    private LocalDateTime birth;
    @ApiModelProperty("身份证号码")
    private String idCard;

    SysPerson(String name, LocalDateTime birth, String idCard) {
        this.name = name;
        this.birth = birth;
        this.idCard = idCard;
    }

    public static com.daoren.graphsqlql.model.entity.SysPerson.SysPersonBuilder builder() {
        return new com.daoren.graphsqlql.model.entity.SysPerson.SysPersonBuilder();
    }

    public String getName() {
        return this.name;
    }

    public LocalDateTime getBirth() {
        return this.birth;
    }

    public String getIdCard() {
        return this.idCard;
    }

    public void setName(String name) {
        this.name = name;
    }

    public void setBirth(LocalDateTime birth) {
        this.birth = birth;
    }

    public void setIdCard(String idCard) {
        this.idCard = idCard;
    }
}
```

可以看到并没有无参构造方法。

正确使用方法：

1. 不要在DO中使用 `@Builder`
2. 如果的确需要使用`@Builder`，请必须加上 `@AllArgsConstructor` 和 `@NoArgsConstructor`~~
